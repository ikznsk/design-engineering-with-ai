## Attributes — 3–5 pairs of opposites

Each pair: a data line it comes from, and the technique it borrows.

### 1. Expressive display type ↔ neutral system type
**Data:** personas.md → Masha, Trust triggers #1: "text reads like a real astrologer, not a template."
**Technique:** CHANI pairs an expressive display face for titles with a plain body face — the wow-moment line and section titles get a face with personality; body copy stays quiet and easy.

### 2. Warm/earthy palette ↔ soft cool pastel palette
**Data:** report.html (already written) describes Masha's own style as "a warm, earthy palette" (Venus in Taurus) — this is a real content line, not a mood board guess. Against that: Glow's soft pink/pale-green/lilac register (jtbd.md Emotional job, "permission to be yourself" — calm, not clinical).
**Technique:** direction options will test both a terracotta/cream/ink register and a lavender/blush/pale-green register — genuinely opposite temperatures, both defensible.

### 3. Editorial density (report) ↔ airy app spacing (onboarding/results)
**Data:** CLAUDE.md → Monetization: the report is the paid deliverable — it has to read as worth paying for, not as a fifth free app screen. Against that: jtbd.md → Emotional job needs onboarding to feel unhurried, not transactional.
**Technique:** borrow professional-report pacing (denser text, generous line-height, section-per-planet) specifically for `report.html`; keep the free-flow screens airier.

### 4. Restrained celestial motif ↔ no illustration at all
**Data:** CLAUDE.md → Critical Product Insight: "the chart shows potential, not the person's current level of development" — an honest product doesn't oversell mysticism. Against that: CHANI shows a few grid-aligned star/celestial marks can still read as warm without reading as fortune-telling.
**Technique:** decide per-direction whether celestial marks appear at all, and if so, how sparingly (a small mark near the chart, never as page-wide decoration).
