## 1. Сутності продукту

| Сутність | Що це | Прив'язаний Job |
|----------|-------|-----------------|
| **Натальна карта** | Розрахована карта по даті/часу/місцю народження | Main Job — основа |
| **Результати (тизер)** | Безкоштовна сторінка: чарт + коротко про нього — wow-момент | Main Job, Emotional job + R1 |
| **Glow Report** | Платний повний репорт — стиль + кар'єра/таланти + енергія + соушал контент (можливо тарифні плани) | Main Job, R1–R4 |
| **Стиль і естетика** | Яка твоя природна візуальна мова — розділ репорту | R1 |
| **Кар'єра і таланти** | Кар'єрне позиціонування, природні таланти й потенціал, де можуть бути затики у виконанні задач — розділ репорту | R2 |
| **Енергія** | Що тебе відновлює і що виснажує — розділ репорту | R3 |
| **Соушал медіа контент** | Який візуал і тип контенту органічний саме для тебе — розділ репорту | R4 |
| **Прев'ю чужих репортів + відгуки** | Соціальний доказ на тизер-сторінці перед покупкою | Конверсія |
| **Wow-момент** | Перша частина тизер-сторінки — щось точне і несподіване | Emotional job + R1 |
| **Онбординг-питання** | "Що зараз відбувається в твоєму житті?" + опціональне друге питання (зміст TBD) | Main Job (тригер) |
| **Shareable картка** | "Мій Glow тип" — для поширення, з безкоштовної тизер-сторінки | Social job |

**Під питанням (без чіткого Job → Backlog):**
- Синастрія / сумісність — є попит, але не в Glow-MVP
- Транзити / прогнози — recurring job не підтверджений для MVP

## 3. Навігаційна модель

**Оновлено 2026-07-17:** глобальної навігації-меню по суті немає — це лінійний funnel, не app. Замість "3 пункти меню" — один шлях зі скролом і одним рішенням (купити чи ні).

| Крок | Що тут | Job |
|------|--------|-----|
| **Тизер-сторінка** | Чарт + wow-момент, скрол до CTA | Main Job, Emotional job |
| **Розблокувати репорт** | Прев'ю + відгуки + checkout | Конверсія |
| **Поділитись** | Shareable Glow картка — доступна прямо з тизер-сторінки, кнопкою, без окремого пункту меню | Social job |

Профіль (5.1/5.2) — іконка в хедері, якщо взагалі є; без persistent навігації сенс мінімальний (див. нотатку в розділі 2).

### Підрахунок тапів — Main Job для Маші

```
Старт (Welcome)
  → тап 1: Введення дати
  → тап 2: Введення часу
  → тап 3: Введення місця
  → тап 4: Питання "що зараз?"
  → тап 5: Додаткове питання (опціонально)
  → [чарт розрахований] → скрол → тап 6: Розблокувати репорт
  → тап 7: Checkout (email + оплата) → Glow Report ✅
```

Онбординг: 5–6 кроків — більше 3, але виправдано: без дати/часу/місця карта неможлива, друге питання опціональне.
На відміну від ранкової версії — тут немає "після першого разу 1 тап", бо немає persistent навігації: людина або вже купила репорт (посилання з email), або проходить онбординг заново.
