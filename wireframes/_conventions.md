# Conventions — Astro Recipe Wireframes

> Ці правила читають усі кроки уроку і уроки 5–8.
> Дата: 2026-07-10 · нав-дерево оновлено 2026-07-19 під funnel-архітектуру

---

## Деталізація

Тільки структура, ієрархія і зони. Сірий, без кольору, шрифтів, бренду і картинок.
Ціль — показати що є на екрані і в якому порядку, а не як це виглядає.

## Розмітка

Семантичний HTML:
- `<header>` — шапка з логотипом і навігацією
- `<nav>` — навігаційна панель-дерево зліва
- `<main>` — основний вміст екрана
- `<section>` — логічна зона з підписом
- `<form>`, `<input>`, `<button>` — справжні елементи форм
- `<article>` — картка або блок контенту

Не використовувати суцільні `<div>` без семантики.

## Текст

Справжній текст з домену Astro Recipe — не lorem ipsum, не "Заголовок 1":
- Імена: Маша, Оля, реальні імена
- Питання онбордингу: реальні формулювання
- Результати: реальні описи ("Твій стиль тяжіє до...")
- CTA: реальні дії ("Розрахувати мій Glow Profile", "Подивитись деталі")
- Помилки: реальний текст ("Місто не знайдено. Спробуй написати інакше або вибери найближче велике місто")

## Назви файлів

Латиниця, через дефіс:
- Базова сторінка (= стан success): `wireframes/<назва>.html`
- Стан empty: `wireframes/<назва>-empty.html`
- Стан error: `wireframes/<назва>-error.html`
- Стан loading: `wireframes/<назва>-loading.html`

Приклад: `results.html`, `results-error.html`, `results-loading.html`

## Стани

Кожен стан — окрема сторінка. Базова сторінка = стан success.
Не робити перемикач/табу на одній сторінці.
Порожній і помилка завжди мають видиму дію-вихід — жодного глухого кута.

## Навігаційна панель

Зліва, однакова на кожній сторінці. Це панель для дизайн-рев'ю/дев-навігації по вайрфреймах, не продуктова навігація — у самому продукті це лінійний funnel без persistent меню (sitemap.md → Навігаційна модель). Дерево:
```
ONBOARDING
  Welcome
  Введення дати
    └─ empty · error
  Введення часу
  Введення місця
    └─ empty · error · loading
  Питання "що зараз?"
  Додаткове питання

РЕЗУЛЬТАТИ (безкоштовний тизер)
  Results
    └─ error · loading

CHECKOUT
  Checkout
    └─ error · loading

GLOW REPORT
  Report

SHARE
  Glow картка
```
Поточна сторінка позначена (bold або підкреслення). Кожен вузол — посилання.

## Що лишаємо на потім

- Колір і градієнти (урок 6)
- Типографіка і шрифти (урок 6)
- Іконки (урок 7)
- Тіні і blur-ефекти (урок 7)
- Анімації і transitions (урок 8)
- Фінальний UI з токенами (уроки 7–8)

## Структура сторінки

```
<body>
  <nav class="wf-nav">   ← дерево зліва, однакове на всіх
  <div class="wf-wrap">
    <header>             ← логотип + крок flow
    <main>
      <section>          ← зони з підписами
    </main>
  </div>
</body>
```

## CSS-принципи

- Фон: #f5f5f5 або білий
- Зони: сіра рамка (#ccc), без кольору
- Кнопки: сірий прямокутник, чорний текст
- Поля вводу: біла рамка, сірий фон
- Підписи зон: маленький сірий текст над зоною
- Шрифт: system-ui або monospace — не завантажувати зовнішні шрифти
